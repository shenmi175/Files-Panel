# 概念说明

## Agent

当前项目里的服务本身就是 agent。

- 它部署在目标服务器上
- 它采集本机资源
- 它操作本机文件
- 它暴露本机 API 和前端界面

## 当前 / 1m / 5m

- `当前`：最近一次采样值
- `1m`：最近 1 分钟窗口的平滑结果
- `5m`：最近 5 分钟窗口的平滑结果

这样做是为了避免把瞬时尖峰误读成长期趋势。

## 趋势图

趋势图读取持久化的历史采样点，而不是仅靠内存缓存。

- 时间范围越大，图表点位越多
- 当点位过多时，后端会降采样
- 图表分辨率不一定等于原始采样周期

## AGENT_ROOT

文件工作区只允许访问 `AGENT_ROOT` 边界内的目录和文件。

它是文件交互的安全边界，不是登录权限本身。

## Docker 状态

Docker 状态依赖当前节点对 `docker.sock` 的访问能力。

如果当前服务用户无法访问 Docker socket，Docker 卡片会显示不可用，而不是伪造空状态。

## 节点目录

节点目录用于登记远程节点元数据：

- 节点名称
- 节点 URL
- 访问令牌
- WireGuard 地址

当前它不是远程控制器，也不是统一代理层。

## WireGuard

WireGuard 更适合作为多节点管理平面的加密网络。

推荐方式：

1. 每台机器部署一个 agent
2. 每台机器加入同一个 WireGuard 网络
3. manager 通过私网访问这些 agent
