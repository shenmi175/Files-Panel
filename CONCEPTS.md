# 概念说明

## Agent

当前服务本身就是 agent：

- 提供前端页面
- 提供 FastAPI API
- 采集本机资源
- 操作本机文件
- 查看本机 Docker / 日志

## 当前 / 1m / 5m

- `当前`：最近一次采样值
- `1m`：最近 1 分钟平滑值
- `5m`：最近 5 分钟平滑值

这样做的目的是把瞬时尖峰和短时趋势分开显示，避免误读。

## 资源历史持久化

资源采样会写入 SQLite。

这意味着：

- 服务重启后历史不会全部丢失
- 可以按时间范围查看趋势
- 图表能做更接近 1Panel / 宝塔的监控体验

## AGENT_ROOT

`AGENT_ROOT` 是文件工作区的根边界。

- 文件列表不能越过它
- 删除、上传、下载、重命名都限制在这个边界内
- 它不等于整台机器根目录

## 本地管理员账号

浏览器登录使用本地管理员账号密码：

- 首次注册一个管理员账号
- 登录成功后由服务端签发会话 Cookie
- 浏览器不再保存 `AGENT_TOKEN` 作为登录凭据

## AGENT_TOKEN

`AGENT_TOKEN` 不是浏览器密码。

它用于：

- 节点目录中的远程节点校验
- 预留的 Bearer 调用场景
- 文件短时签名下载

## 节点目录

节点目录当前主要保存：

- 节点名称
- 节点 URL
- 节点 Token
- WireGuard 地址

它现在是多节点管理的准备层，不代表已经具备完整远程执行能力。

## WireGuard

WireGuard 当前适合作为管理平面网络：

1. 每台目标机器部署一个 agent
2. 节点通过 WireGuard 私网互通
3. 后续 manager / proxy 层再通过这些私网地址访问各 agent
